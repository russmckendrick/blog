Monitoring Docker covers how to keep an eye on Docker hosts, containers, and the workloads running inside them. The book starts with Docker's own stats commands, then layers in third-party tooling: cAdvisor, SysDig, Prometheus, and Datadog, alongside log shipping to a central service. Each chapter lands on a specific tool and a worked example, and there's a chapter on tying container monitoring into an existing Zabbix install for teams that already run one.

Aimed at sysadmins and DevOps engineers who already use Docker and want to understand what is happening inside it once it is running in production.
